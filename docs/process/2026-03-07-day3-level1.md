# Day 3 Level 1 Log (2026-03-07)

## Objective
Ship a runnable Level 1 deterministic pipeline with the first explicit connector:
- detection -> segmentation prompts -> segmentation masks

## What Was Built
1. Added `YOLOv8Stage` (`src/cvpo/stages/detection.py`):
   - deterministic threshold-based detection backend
   - optional real YOLO backend via `ultralytics`
2. Added `SAM2Stage` (`src/cvpo/stages/segmentation.py`):
   - deterministic mask generation from prompt boxes
   - explicit SAM2 backend hook with clear not-yet-wired message
3. Added connector:
   - `DetectionToSegmentationConnector` in `src/cvpo/connectors/detection_to_segmentation.py`
   - converts `DetectionResult` into `SegmentationPromptSet`
4. Added Level 1 pipeline builder:
   - `src/cvpo/pipelines/level1.py`
5. Added CLI demo:
   - `cvpo --level1-demo`
6. Extended typed contracts:
   - `SegmentationPrompt`, `SegmentationPromptSet`

## Why This Matters
This is the first end-to-end deterministic multi-stage pipeline in CVPO with an explicit adapter.
It validates the core architectural claim: composable stage chaining via typed connectors.

## Validation
- New tests added for:
  - connector behavior
  - level1 pipeline execution
  - CLI level1 output
